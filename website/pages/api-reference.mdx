# API Reference

## `dts-cli watch`

```sh
Description
  Rebuilds on any change

Usage
  $ dts-cli watch [options]

Options
  -i, --entry           Entry module(s)
  --target              Specify your target environment  (default web)
  --name                Specify name exposed in UMD builds
  --format              Specify module format(s)  (default cjs,esm)
  --tsconfig            Specify your custom tsconfig path (default <root-folder>/tsconfig.json)
  --verbose             Keep outdated console output in watch mode instead of clearing the screen
  --onFirstSuccess      Run a command on the first successful build
  --onSuccess           Run a command on a successful build
  --onFailure           Run a command on a failed build
  --noClean             Don't clean the dist folder
  --transpileOnly       Skip type checking
  -h, --help            Displays this message

Examples
  $ dts-cli watch --entry src/foo.tsx
  $ dts-cli watch --target node
  $ dts-cli watch --name Foo
  $ dts-cli watch --format cjs,esm,umd
  $ dts-cli watch --tsconfig ./tsconfig.foo.json
  $ dts-cli watch --noClean
  $ dts-cli watch --onFirstSuccess "echo The first successful build!"
  $ dts-cli watch --onSuccess "echo Successful build!"
  $ dts-cli watch --onFailure "echo The build failed!"
  $ dts-cli watch --transpileOnly
```

## `dts-cli build`

```sh
Description
  Build your project once and exit

Usage
  $ dts-cli build [options]

Options
  -i, --entry           Entry module(s)
  --target              Specify your target environment  (default web)
  --name                Specify name exposed in UMD builds
  --format              Specify module format(s)  (default cjs,esm)
  --extractErrors       Opt-in to extracting invariant error codes
  --tsconfig            Specify your custom tsconfig path (default <root-folder>/tsconfig.json)
  --transpileOnly       Skip type checking
  -h, --help            Displays this message

Examples
  $ dts-cli build --entry src/foo.tsx
  $ dts-cli build --target node
  $ dts-cli build --name Foo
  $ dts-cli build --format cjs,esm,umd
  $ dts-cli build --extractErrors
  $ dts-cli build --tsconfig ./tsconfig.foo.json
  $ dts-cli build --transpileOnly
```

## `dts-cli test`

This runs Jest v24.x, forwarding all CLI flags to it. See [https://jestjs.io](https://jestjs.io) for options. For example, if you would like to run in watch mode, you can run `dts-cli test --watch`. So you could set up your `package.json` `scripts` like:

```json
{
  "scripts": {
    "test": "dts-cli test",
    "test:watch": "dts-cli test --watch",
    "test:coverage": "dts-cli test --coverage"
  }
}
```

## `dts-cli lint`

```sh
Description
  Run eslint with Prettier

Usage
  $ dts-cli lint [options]

Options
  --fix               Fixes fixable errors and warnings
  --ignore-pattern    Ignore a pattern
  --write-file        Write the config file locally
  --report-file       Write JSON report to file locally
  -h, --help          Displays this message

Examples
  $ dts-cli lint src
  $ dts-cli lint src --fix
  $ dts-cli lint src test --ignore-pattern test/foo.ts
  $ dts-cli lint src --write-file
  $ dts-cli lint src --report-file report.json
```
